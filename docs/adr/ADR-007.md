# ADR-007: Angular and Angular Material for the Admin Interface

## 1. Context

The platform requires a separate, secure, and highly functional web application for administrators and moderators. The primary purpose of this interface is data management, such as reviewing moderation queues, searching for users or content, and viewing system analytics. The UI must be robust, data-driven, and optimized for productivity on desktop devices.

## 2. Decision

We will build the admin interface as a standalone Single-Page Application (SPA) using **Angular**. The UI will be built with the **Angular Material** component library, which offers a comprehensive set of data-centric components well-suited for administrative dashboards.

## 3. Status

Accepted.

## 4. Consequences

### Positive ✅

*   **Optimized for Data-Driven Applications**: Angular's opinionated structure, built-in form handling, and powerful dependency injection system make it exceptionally well-suited for building complex, data-heavy enterprise applications like an admin dashboard.
*   **Mature Component Library**: Angular Material provides high-quality, accessible, and feature-rich components like sortable/filterable data tables (`MatTable`), paginators, and complex form controls that are essential for an admin UI and would be time-consuming to build from scratch.
*   **Strict Separation of Concerns**: Having a completely separate front-end application for administration enforces a strong security boundary. The admin UI can be deployed and protected independently from the public-facing application, and it communicates with the backend exclusively through a secured API.
*   **Developer Experience**: The Angular CLI provides excellent tooling for scaffolding, building, and testing the application, leading to a structured and maintainable codebase.

### Negative ❌

*   **Polyglot Overhead**: This decision introduces a second front-end framework and language (TypeScript) into the stack, as noted in [ADR-001](./ADR-001.md). This requires maintaining a separate toolchain and knowledge base. However, this is a deliberate choice to use the best tool for a specific job (data-heavy admin UI vs. public-facing C# PWA).
*   **Learning Curve**: Angular has a steeper learning curve compared to some other frameworks. This is not a significant risk as the developer has proficiency in it, and its complexity is justified by the requirements of the admin application.
